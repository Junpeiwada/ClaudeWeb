# ユニバーサルバイナリ対応

## 現状

現在のリリースは**ビルドしたMacのアーキテクチャ専用**のバイナリを生成する。

- Arm Mac（M1/M2/M3/M4）でビルド → `aarch64-apple-darwin` 専用
- Intel Macでビルド → `x86_64-apple-darwin` 専用

### 影響を受ける箇所

| 箇所 | 内容 |
|------|------|
| Tauriバイナリ（Rust） | `npx tauri build` がホストアーキテクチャ向けにコンパイルする |
| sharpネイティブモジュール | `build-server.mjs` でアーキテクチャ別パッケージをコピー（`@img/sharp-darwin-arm64` / `@img/sharp-darwin-x64`）。存在するもののみコピーされる |
| latest.json | `release.sh` が `darwin-aarch64` と `darwin-x86_64` の両方に同じバイナリURLを記載している。実際は片方のアーキテクチャしか動作しない |

## ユニバーサルバイナリ対応の手順（未実施）

両アーキテクチャ対応のバイナリを作る場合、以下の対応が必要。

### 1. Rustクロスコンパイルターゲットの追加

Arm Macで作業する場合：

```bash
rustup target add x86_64-apple-darwin
```

Intel Macで作業する場合：

```bash
rustup target add aarch64-apple-darwin
```

### 2. Tauriビルドコマンドの変更

```bash
npx tauri build --target universal-apple-darwin
```

これにより `aarch64` と `x86_64` の両方をコンパイルし、macOSのユニバーサルバイナリ（fat binary）として結合される。

### 3. sharpの両アーキテクチャバンドル

`build-server.mjs` の `packagesToCopy` には既に両方記載済み：

```js
const packagesToCopy = [
  "@anthropic-ai/claude-agent-sdk",
  "@img/sharp-darwin-arm64",
  "@img/sharp-darwin-x64",
];
```

ただし `npm install` は通常ホストアーキテクチャのパッケージのみインストールする。両方をバンドルするには：

```bash
npm install --cpu=arm64 @img/sharp-darwin-arm64
npm install --cpu=x64 @img/sharp-darwin-x64
```

### 4. release.shの変更

ユニバーサルバイナリの場合、成果物のパスが変わる：

- 現在: `src-tauri/target/release/bundle/`
- 変更後: `src-tauri/target/universal-apple-darwin/release/bundle/`

`release.sh` 内の `BUNDLE_DIR` を更新する必要がある。

### 5. latest.jsonの整合性

ユニバーサルバイナリであれば、`darwin-aarch64` と `darwin-x86_64` に同じURLを指定する現在の構成で正しく動作する。

## 備考

- Rosetta 2が有効なArm Macであれば、x86_64バイナリも動作する（パフォーマンスは劣る）
- 逆にIntel Macではarm64バイナリは動作しない
- 現在はArm Macでリリースしているため、Intel Macユーザーは実質サポート外
