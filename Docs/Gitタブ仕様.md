# Gitタブ仕様

## 概要

チャット・ファイルタブに続く3番目のタブとして「Git」タブを追加する。
GitHub Desktopライクなインターフェースで、基本的なGit操作（ステージング、コミット、Fetch/Pull/Push）とコミット履歴の閲覧を提供する。

## サブタブ構成

Gitタブ内に **Changes** / **History** の2つのサブタブを持つ。ヘッダーバーの直下に配置し、クリックで切り替える。

- **Changes**: ワーキングツリーの変更ファイル一覧、ステージング、コミット操作（従来の機能）
- **History**: コミット履歴の一覧表示と、各コミットの詳細・差分表示

### レスポンシブ対応

- デスクトップ（768px以上）: 左右2ペイン（ファイル一覧:Diff = 3:7 程度）
- モバイル（768px未満）: 上下スタック（ファイル一覧 → タップでDiff表示に切り替え）

## ヘッダーバー

### ブランチ表示

- 現在のブランチ名を表示（読み取り専用、切り替え・作成機能なし）
- アイコン: Gitブランチアイコン
- ブランチ名が長い場合は省略表示（ellipsis）

### Fetch / Pull / Push ボタン

状態に応じてボタンの表示が動的に切り替わる（GitHub Desktop方式）。

| 状態 | ボタン表示 | 動作 |
|------|-----------|------|
| 初期状態 / リモートと同期済み | **Fetch origin** | `git fetch origin` を実行 |
| fetch後、リモートが先行 | **Pull origin (N)** | `git pull --ff-only origin <branch>` を実行 |
| ローカルが先行（unpushed commits） | **Push origin (N)** | `git push origin <branch>` を実行 |

- **(N)** はコミット数の差分を表示
- ボタンは1つのみ表示（3つ並べない）
- 操作中はローディングスピナーを表示し、ボタンを無効化

#### Pull の戦略

2段階で試行する:

1. まず `git pull --ff-only` を試行
2. fast-forward できない場合（ブランチが分岐している場合）→ `git pull --rebase` を試行
3. rebase 中にコンフリクトが発生した場合 → `git rebase --abort` で即座に取り消し

これにより、**コンフリクトが起きない限りリベースで自動解決**し、**コンフリクトが起きたら元の状態に完全復元**する。

- コンフリクト時のエラーメッセージ: 「コンフリクトが発生したためリベースを中止しました。ターミナルで手動解決してください。」
- **ワーキングツリーが壊れた状態で放置されることはない**

#### Push の安全策

- リモートがローカルより先行している場合、Pushボタンは表示しない（先にPullを促す）
- Push失敗時はエラーメッセージを表示

## 変更ファイル一覧（左ペイン）

### 表示内容

各ファイルに以下を表示:

- **チェックボックス**: ステージ状態の管理
- **ステータスバッジ**: ファイルの変更種別
- **ファイルパス**: リポジトリルートからの相対パス

### ステータスバッジ

| バッジ | 意味 | 色 |
|--------|------|-----|
| M | Modified（変更） | オレンジ |
| A | Added（新規追加・tracked） | 緑 |
| D | Deleted（削除） | 赤 |
| R | Renamed（リネーム） | 青 |
| ? | Untracked（未追跡） | グレー |

### ステージング操作

- チェックボックスON → `git add <file>`
- チェックボックスOFF → `git reset HEAD <file>`（unstage）
- 未追跡ファイル（?）のチェックON → `git add <file>`
- ヘッダーに「全選択 / 全解除」のチェックボックスを設置
- **初期状態**: 変更ファイルはすべてチェックON（ステージ済み）の状態で表示

### ファイル選択とDiff表示

- ファイルをクリック → 右ペインにそのファイルのDiffを表示
- 選択中のファイルはハイライト表示
- 最初に表示した時点で、一覧の先頭ファイルを自動選択

### 空状態

変更ファイルがない場合:
- ファイル一覧エリアに「変更はありません」のメッセージを中央表示

## Diff表示（右ペイン）

### 形式

- Unified diff形式（1列表示）
- シンタックスハイライトなし（シンプルな +/- 色分けのみ）

### 色分け

| 行の種類 | 背景色 | テキスト色 |
|----------|--------|-----------|
| 追加行（+） | 薄い緑 | 緑 |
| 削除行（-） | 薄い赤 | 赤 |
| コンテキスト行 | なし | 通常テキスト色 |
| ハンク行（@@） | 薄い青 | 青 |

### 新規ファイル / 削除ファイル

- 新規ファイル: 全行が追加行として表示
- 削除ファイル: 全行が削除行として表示
- バイナリファイル: 「バイナリファイルです。差分は表示できません。」のメッセージ

### 空状態

ファイルが未選択の場合:
- 「ファイルを選択してください」のメッセージを中央表示

## コミット（左ペイン下部）

### コミットメッセージ

- 単一行テキストフィールド（タイトルのみ）
- プレースホルダー: 「コミットメッセージを入力」
- 最大文字数制限なし（UI上は1行表示で、長い場合はスクロール）

### コミットボタン

- ラベル: 「**コミット (N files)**」（Nはチェックされたファイル数）
- チェックされたファイルが0件の場合: ボタンを無効化
- コミットメッセージが空の場合: ボタンを無効化
- 実行する操作: ステージ済みファイルに対して `git commit -m "<message>"`

### コミット成功後

- コミットメッセージフィールドをクリア
- ファイル一覧を再取得して更新
- ヘッダーのボタンが「Push origin (1)」に切り替わる

## コミット履歴（History）

### 左ペイン: コミット一覧

- 各コミットに以下を表示:
  - **コミットメッセージ**（1行目のみ）
  - **著者名**
  - **相対時間**（「2時間前」「1日前」等。30日以上前は日付表示）
- 選択中のコミットはハイライト表示
- 初期表示: 最新100件をロード
- 末尾に「**さらに読み込む**」ボタンを配置し、100件ずつ追加ロード
- 空状態: 「コミット履歴がありません」

### 右ペイン: コミット詳細

選択したコミットの詳細情報と差分を表示する。

#### コミット情報ヘッダー

- コミットメッセージ全文（複数行対応）
- 著者名、短縮ハッシュ（7文字）、日時

#### ファイルごとのDiffアコーディオン

コミットに含まれる変更ファイルをファイル単位でアコーディオン表示する。

- 各ファイルのヘッダーに以下を表示:
  - **開閉アイコン**（▶ 閉じ / ▼ 開き）
  - **ステータスバッジ**（M/A/D/R、Changesと同じ色）
  - **ファイルパス**
  - **追加行数**（+N、緑）/ **削除行数**（-N、赤）
- ヘッダークリックでdiffの開閉を切り替え
- **初期状態: すべて閉じた状態**
- Diff本体の色分けはChangesビューと同一仕様

#### 空状態

コミット未選択時: 「コミットを選択してください」

### モバイル対応

- コミットリスト → タップ → コミット詳細画面に切り替え（Changesビューと同じパターン）
- 戻るボタン（←）で「コミット一覧に戻る」

## API設計

### エンドポイント一覧

すべてのエンドポイントはプレフィックス `/api/repos/:repoId/git/` の下に配置。

| メソッド | パス | 説明 |
|---------|------|------|
| GET | `/status` | ブランチ名、変更ファイル一覧、ahead/behind数を取得 |
| GET | `/diff?file=<path>` | 指定ファイルのunified diffを取得 |
| GET | `/log?page=1&perPage=100` | コミット履歴一覧（ページネーション対応） |
| GET | `/show?commit=<hash>` | コミットの詳細（メッセージ全文、変更ファイル、diff） |
| POST | `/stage` | ファイルをステージ（add）する |
| POST | `/unstage` | ファイルをアンステージ（reset）する |
| POST | `/commit` | ステージ済みファイルをコミット |
| POST | `/fetch` | `git fetch origin` を実行 |
| POST | `/pull` | `git pull --ff-only` を実行 |
| POST | `/push` | `git push origin <branch>` を実行 |

### GET `/status` レスポンス

```json
{
  "branch": "main",
  "ahead": 0,
  "behind": 0,
  "files": [
    {
      "path": "src/app.ts",
      "status": "M",
      "staged": true
    },
    {
      "path": "newfile.ts",
      "status": "?",
      "staged": false
    }
  ]
}
```

- `ahead`: ローカルがリモートより先行しているコミット数
- `behind`: リモートがローカルより先行しているコミット数
- `staged`: 現在のステージ状態

### GET `/log` レスポンス

```json
{
  "commits": [
    {
      "hash": "abc1234def5678...",
      "message": "fix: バグ修正",
      "author": "Junpei Wada",
      "date": "2026-04-02T15:30:00+09:00"
    }
  ],
  "hasMore": true
}
```

- `page`: ページ番号（1始まり）
- `perPage`: 1ページあたりの件数（デフォルト100、最大200）
- `hasMore`: 次のページが存在するか

### GET `/show` レスポンス

```json
{
  "hash": "abc1234def5678...",
  "message": "fix: バグ修正\n\n詳細な説明...",
  "author": "Junpei Wada",
  "date": "2026-04-02T15:30:00+09:00",
  "files": [
    { "path": "src/app.ts", "status": "M" },
    { "path": "newfile.ts", "status": "A" }
  ],
  "diff": "diff --git a/src/app.ts b/src/app.ts\n..."
}
```

- `commit` パラメータ: 4〜40文字の16進数ハッシュ（バリデーションあり）
- `message`: コミットメッセージ全文（Body含む）
- `files`: 変更ファイル一覧（`diff-tree --root --name-status` で取得、ルートコミット対応）
- `diff`: unified diff全文（`diff-tree --root -p` で取得）
- タイムアウト: diff取得は30秒

### POST `/stage` リクエスト

```json
{
  "files": ["src/app.ts", "newfile.ts"]
}
```

### POST `/unstage` リクエスト

```json
{
  "files": ["src/app.ts"]
}
```

### POST `/commit` リクエスト

```json
{
  "message": "コミットメッセージ"
}
```

### POST `/fetch`, `/pull`, `/push`

リクエストボディなし。レスポンスは操作結果のみ。

```json
{
  "success": true,
  "message": "Fetched successfully"
}
```

エラー時:

```json
{
  "success": false,
  "error": "fast-forwardできません。ブランチが分岐しています。"
}
```

## エラーハンドリング

### エラー表示方法

- 操作失敗時は画面上部にSnackbar（トースト通知）で表示
- 赤色背景、自動消去（5秒後）、手動閉じ可能

### 想定されるエラーケース

| 操作 | エラー | メッセージ |
|------|--------|-----------|
| Pull | rebaseでコンフリクト | 「コンフリクトが発生する操作のため、Pullを中止しました。リポジトリは元の状態に戻っています。ターミナルで手動解決してください。」 |
| Pull | ローカル変更と競合 | 「ローカルの変更がPullにより上書きされるため中止されました。先にコミットするか、ターミナルでstashしてください。」 |
| Push | リモートが先行 | 「リモートに新しいコミットがあります。先にPullしてください。」 |
| Push | 認証エラー | 「認証に失敗しました。Git認証設定を確認してください。」 |
| Commit | ステージファイルなし | UI上でボタンを無効化（エラーにならない） |
| 全般 | Gitリポジトリでない | 「このディレクトリはGitリポジトリではありません。」（タブ自体を無効化） |

### コンフリクト安全策

**Pullの安全性（2段階戦略）:**
1. まず `--ff-only` を試行。fast-forwardできればそのまま完了（コンフリクトは原理的に起きない）
2. fast-forwardできない場合、`--rebase` を試行。コンフリクトなしなら完了
3. rebaseでコンフリクト発生 → `git rebase --abort` で即座に元の状態に復元

**ローカル未コミット変更がある場合:**
- Pullで更新されるファイルとローカル変更が競合 → Gitが操作を拒否。ワーキングツリーは変更されない

いずれの場合も:
1. ワーキングツリーは安全に保たれる
2. ユーザーにエラーメッセージで通知
3. ターミナルでの手動操作を案内

## フロントエンド実装方針

### ファイル構成

```
frontend/src/
├── pages/
│   └── GitPage.tsx              # Gitタブのメインページ（Changes/History切り替え）
├── components/
│   └── git/
│       ├── GitHeader.tsx        # ブランチ表示 + Fetch/Pull/Push ボタン
│       ├── GitSubTabs.tsx       # Changes/History サブタブ切り替え
│       ├── GitFileList.tsx      # 変更ファイル一覧 + ステージングチェックボックス
│       ├── GitDiffView.tsx      # Diff表示エリア（Changesビュー）
│       ├── GitCommitBox.tsx     # コミットメッセージ + コミットボタン
│       ├── GitCommitList.tsx    # コミット履歴一覧（Historyビュー左ペイン）
│       └── GitCommitDetail.tsx  # コミット詳細 + ファイル別アコーディオンdiff（Historyビュー右ペイン）
└── hooks/
    ├── useGitStatus.ts          # Changes用の状態管理フック
    └── useGitHistory.ts         # History用の状態管理フック（履歴取得、ページネーション、詳細取得）
```

### ルーティング追加

```
/:repo (RootLayout)
  ├── /chat
  ├── /chat/:sessionId
  ├── /files/*
  └── /git              ← 新規追加
```

### タブ追加

RootLayoutのtabs配列に追加:

```tsx
{ key: "git", label: "Git", icon: <GitIcon /> }
```

### 状態管理

- **Changesビュー**: Gitタブを開いた時に `/status` を1回呼び出してファイル一覧を取得。ステージング操作後、コミット・Fetch・Pull・Push後も再取得
- **Historyビュー**: Historyサブタブに初めて切り替えた時に `/log` を1回呼び出して最新100件を取得。コミット選択時に `/show` で詳細を取得
- 自動ポーリングなし（手動リフレッシュのみ）

### マウント方針（Chatの常時マウントを維持する）

**重要: Chatコンポーネントは常時マウントを維持すること。**

現在の RootLayout の構造:
- Chat → `display: none/flex` で表示切り替え（常時マウント、SSE接続を維持）
- Files → `<Outlet />` 経由（タブ切り替えでアンマウント）

Gitタブ追加後の構造:

```tsx
{/* Chat（常にマウント、display で切り替え） */}
<Box sx={{ display: activeTab === "chat" ? "flex" : "none" }}>
  <Chat ... />
</Box>

{/* Files / Git（Outlet経由、ルーティングで排他的に切り替え） */}
<Box sx={{ display: activeTab !== "chat" ? "flex" : "none" }}>
  <Outlet />
</Box>
```

- Git と Files はどちらも `<Outlet />` を通じてレンダリングされ、ルーティングにより排他的に切り替わる
- Chat は他のタブがアクティブでも常にDOMに存在し、SSEストリーミング接続が途切れない
- Gitタブは常時接続の必要がなく、タブ切り替えごとに最新状態を取得する方が適切なため、Outlet経由で問題ない

## バックエンド実装方針

### ファイル構成

```
server/
└── routes/
    └── git.ts           # 全Gitエンドポイントを集約
```

### Gitコマンド実行

- Node.jsの `child_process.execFile` を使用（シェル経由しない）
- コマンドインジェクション防止のため、引数は配列で渡す
- 実行ディレクトリ（cwd）はリポジトリのフルパスを指定
- タイムアウト: 30秒（Fetch/Pull/Push）、10秒（status/diff/stage/unstage/commit）

### セキュリティ

- ファイルパスのバリデーション: `..` を含むパスは拒否
- リポジトリパスの存在確認: `.git` ディレクトリの存在をチェック
- コマンド引数のサニタイズ: `execFile` による自動エスケープに依存
