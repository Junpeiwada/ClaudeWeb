## Phase 2: バックエンド API 基盤

Express サーバと各 API エンドポイントのスケルトンを実装。

- [x]2-1. `server/index.ts` — Express サーバ起動・ルーティング登録
- [x]2-2. `GET /api/repos` — `$BASE_PROJECT_DIR/` 直下のディレクトリを動的スキャンして返す
- [x]2-3. `GET /api/status` — セッション状態を返すエンドポイント（スケルトン）
- [x]2-4. `POST /api/chat` — SSE ストリーミングのエンドポイント（スケルトン、まずモック応答）
- [x]2-5. `POST /api/permission` — 権限承認エンドポイント（スケルトン）

**完了条件**: 各エンドポイントに curl でリクエストして正常なレスポンスが返る

## Phase 3: Claude Code SDK 連携

実際に Claude Code SDK を呼び出し、SSE でストリーミング返却する。

- [x]3-1. `server/claude/executor.ts` — Claude Code SDK ラッパー実装
  - 新規セッション開始（`prompt` + `cwd`）
  - セッション継続（`sessionId` 引き継ぎ）
  - ストリーミングイベントのコールバック処理
- [x]3-2. `POST /api/chat` にSDK連携を組み込み、テキスト応答を SSE でストリーム
- [x]3-3. 権限リクエスト（permission request）のハンドリング
  - SDK からの権限要求を検出 → SSE で `type: "permission"` イベント送信
  - `POST /api/permission` で承認/拒否を受け取り SDK に応答
- [x]3-4. シングルセッション管理
  - 進行中セッションの追跡
  - 新規リクエスト時に既存セッションを abort
  - `GET /api/status` で現在のセッション状態を返す

**完了条件**: curl で `/api/chat` にメッセージを送り、Claude Code の応答が SSE でストリーミングされる
